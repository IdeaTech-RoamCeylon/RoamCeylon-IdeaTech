# Machine Learning Feature Engineering Plan
**Phase:** Month 4 - ML Transition Architecture

## 1. Feature Engineering Planning
To transition from static rule-based heuristics to a predictive Machine Learning ranking model, we must translate our raw relational database records into standardized numerical and categorical features. 

The pipeline will extract data based on the following concepts:
* **User Interest Profile:** Translating explicit profile settings (likes/dislikes) into a weighted vector.
* **Preference Patterns:** Identifying sequential behaviors (e.g., users who select "Culture" often select "Food" next).
* **Feedback Ratings:** Normalizing 1-5 star ratings into a continuous, weighted satisfaction score.
* **Trip Category Usage:** Tracking the historical distribution of categories in a user's accepted itineraries to identify hidden preferences.

---

## 2. Defined ML Features (Data Dictionary)
The following schemas define the exact features that will be extracted, normalized, and fed into the Month 4 ML training pipeline.

### A. User Features (User State)
These features represent the individual user's historical behavior and explicit settings.
* `preferred_categories` *(Vector)*: A weighted array representing the user's affinity for each category (e.g., `[Nature: 0.8, Culture: 0.2, Food: 0.5]`).
* `feedback_positivity_rate` *(Float, 0.0 - 1.0)*: The historical percentage of the user's feedback that is 4 stars or higher.
* `travel_pace_preference` *(Categorical/Ordinal)*: The user's preferred itinerary density (`0` = Relaxed, `1` = Moderate, `2` = Packed).

### B. Destination Features (Item State)
These features represent the static and globally crowdsourced attributes of the locations themselves.
* `popularity_score` *(Float, 0.0 - 1.0)*: A normalized score based on how frequently this destination is included in saved trips across the entire platform.
* `category_type` *(Categorical)*: The primary classification of the location (e.g., 'culture', 'nature', 'food').
* `aggregate_feedback_rating` *(Float, 1.0 - 5.0)*: The globally averaged star rating for this specific destination.

### C. Interaction Features (Contextual/Behavioral State)
These features represent the specific relationship between a particular user and a particular destination.
* `click_frequency` *(Integer)*: The total number of times the user has clicked on this destination's detail card to learn more.
* `trip_selection_rate` *(Float, 0.0 - 1.0)*: The ratio of times this destination was kept in a finalized trip versus how many times it was rejected/regenerated by the user.
